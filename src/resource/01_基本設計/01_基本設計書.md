# ブロック崩しゲーム 基本設計書

## 1. システム概要

本システムは、Javaの学習題材として開発するデスクトップ向けブロック崩しゲームである。  
プレイヤーはパドルを左右に操作し、ボールを跳ね返して画面上部のブロックを破壊する。  
すべてのブロックを破壊するとゲームクリア、ボールを落として残機が0になるとゲームオーバーとする。

## 2. 開発目的

| 項目 | 内容 |
|---|---|
| 主目的 | Javaによるオブジェクト指向設計、イベント処理、描画処理の学習 |
| 学習対象 | クラス設計、ゲームループ、衝突判定、キーボード入力、Swing描画 |
| 対象外 | ネットワーク通信、DB保存、オンラインランキング、高度な物理演算 |

## 3. 前提条件

| 項目 | 内容 |
|---|---|
| 実行環境 | PC上で動作するデスクトップアプリケーション |
| 使用言語 | Java 17以上 |
| GUI | Swing |
| 描画 | Java2D |
| データ保存 | なし |
| 外部ライブラリ | 原則使用しない |

## 4. 機能一覧

| 機能ID | 機能名 | 概要 |
|---|---|---|
| F-01 | ゲーム画面表示 | ゲームウィンドウを表示する |
| F-02 | パドル操作 | 左右キーでパドルを移動する |
| F-03 | ボール移動 | ボールを一定周期で移動させる |
| F-04 | 壁衝突判定 | ボールが壁に当たった場合に反射する |
| F-05 | パドル衝突判定 | ボールがパドルに当たった場合に反射する |
| F-06 | ブロック衝突判定 | ボールがブロックに当たった場合にブロックを消す |
| F-07 | スコア管理 | ブロック破壊時にスコアを加算する |
| F-08 | 残機管理 | ボールを落とした場合に残機を減らす |
| F-09 | ゲーム状態管理 | 開始前、プレイ中、一時停止、クリア、ゲームオーバーを管理する |
| F-10 | リスタート | ゲーム終了後に再開始できる |

## 5. 非機能要件

| 区分 | 要件 |
|---|---|
| 操作性 | キーボードのみで操作できること |
| 保守性 | ボール、パドル、ブロックなどをクラス分割すること |
| 可読性 | 学習用のため、過度な抽象化を避けること |
| 性能 | 60FPS程度で自然に動作すること |
| 拡張性 | ステージ数、ブロック配置、スコア計算を後から変更しやすいこと |

## 6. 画面仕様

### 6.1 ゲーム画面

| 表示要素 | 内容 |
|---|---|
| スコア | 現在の得点 |
| 残機 | 残りのボール数 |
| ボール | 自動で移動する円 |
| パドル | プレイヤーが操作するバー |
| ブロック | 破壊対象 |
| メッセージ | START、PAUSE、GAME OVER、CLEARなど |

### 6.2 操作仕様

| キー | 処理 |
|---|---|
| ← | パドルを左へ移動 |
| → | パドルを右へ移動 |
| Space | ゲーム開始 / リスタート |
| P | 一時停止 / 再開 |
| Esc | 終了 |

## 7. ゲーム状態

| 状態 | 説明 |
|---|---|
| READY | 開始前 |
| PLAYING | プレイ中 |
| PAUSED | 一時停止中 |
| GAME_OVER | ゲームオーバー |
| CLEAR | ゲームクリア |

## 8. ゲームルール

| 項目 | 内容 |
|---|---|
| 初期残機 | 3 |
| ブロック破壊点 | 10点 |
| クリア条件 | すべてのブロックを破壊 |
| ゲームオーバー条件 | 残機が0になる |
| ボール落下判定 | ボールのY座標が画面下端を超えた場合 |

## 9. 拡張候補

| 拡張 | 学べること |
|---|---|
| ステージ制 | データ構造、初期化処理 |
| ブロックの耐久値 | 状態管理 |

| 効果音 | リソース管理 |
